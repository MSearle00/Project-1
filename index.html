Github Workflow

As shown in the image above, the Github workflow is branch-based and supports projects that have regular depoloyments.

  1. A branch is an isolated set of code, which is under version control, meaning modifications do not alter the master branch and can happen alongside each other in separate branches. These branches get merged and added to the master once they are complete and tested.
  2. A commit is a change to a file.
  3. A pull request is essentially a request for a review. It allows people to view your commits, discuss any modifications if needed and submit their own commits if necessary.

Unix Shell

The Unix shell is a command line interpreter. Bash is an example of a Unix shell and a command language.

  • ls lists files in your current directory
  • pwd print working directory, shows you your current working directory
  • cd changes your directory
  • mkdir make directory, creates a new directory
  • touch creates an empty text file
  • rm removes files
